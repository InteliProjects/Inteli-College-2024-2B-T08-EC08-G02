## **1.0** Introdução

Um **mockup** é uma representação visual detalhada de uma interface de usuário, projetada para apresentar a aparência e o design de um sistema ou aplicativo, incluindo elementos como cores, tipografia e estilo. Ele ajuda a visualizar o layout final e a estética do produto, fornecendo uma visão mais realista de como o sistema será percebido pelos usuários. Este documento descreve o mockup de um sistema de gestão hospitalar, abrangendo telas principais e modais essenciais para médicos e enfermeiros, oferecendo uma base clara para validação do design e feedback antes do desenvolvimento final.[1]

## **2.0** Telas do Mockup

## **2.1** Tela de Login

![Login](/img/ux/Mockup/Login.png)
A tela de login permite que os usuários insiram suas credenciais para acessar o sistema. Os campos principais incluem:

- Campo de usuário (e-mail ou ID).
- Campo de senha.
- Botão para acessar o sistema.
- Link para recuperação de senha.

## **2.2** Tela inicial do médico

![Home - Doctor](/img/ux/Mockup/Home_Doctor.png)
A tela inicial do médico apresenta:

- Acesso rápido as informações dos pacientes em tratamento registrados e monitorados por aquele médico.
- Notificações e alertas de novos registros ou atualizações importantes.

## **2.3** Tela de infomações do paciente

![Pacient Description](/img/ux/Mockup/Pacient_Description.png)
Esta tela fornece uma visão detalhada do paciente selecionado, incluindo:

- Informações pessoais.
- Histórico de saúde.
- Tratamentos em andamento.
- Repostas para as perguntas feitas pelo Robô.
- Últimas atualizações colhidas pelo Robô.


## **2.4** Modal de registro de paciente

![New Patient Modal](/img/ux/Mockup/New_Patient_Modal.png)
Este modal permite a adição de um novo paciente. Em que o médico precisa colocar o indentificador do paciente desejado para começar a fazer o seu monitoramento, através da plataforma.



## **2.5** Tela inicial do enfermeiro

![Home - Nurse](/img/ux/Mockup/Home_Nurse.png)
A tela inicial do enfermeiro contém:

- Listagem de todos pacientes em tratamento no hospital.
- Notificações de pacientes que precisam de atenção especial.

## **2.6** Tela inicial de registro do paciente

![Patient Record Modal](/img/ux/Mockup/Patient_Record_Modal.png)
Este modal é utilizado para registrar dados e informações do paciente, incluindo:

- Entrada de dados vitais (temperatura, pressão, etc.).
- Anotações sobre o estado do paciente.
- Opções para salvar ou descartar as atualizações.

## **2.7** Tela de robôs para o enfermeiro

![Robots - Nurse](/img/ux/Mockup/Robots_Nurse.png)
Tela específica para controle e monitoramento de robôs assistentes utilizados pelo médico. Inclui:

- Status dos robôs.
- Funções específicas atribuídas a cada robô.
- Alertas e notificações relacionadas ao funcionamento dos robôs.

## **2.8** Modal de atribuição de pacientes para Robôs

![Patient Record Modal-1](/img/ux/Mockup/Robot_Calendar.png)
Graças a esse modal é possível atribuir um paciente com um horário especifico para que o robo faça seu monitoramento e com isso consiga organizar quando cada paciente vai ser atendido pela solução.

---

## **3.0** Conclusão

Este mockup documenta o design detalhado e o layout do sistema, fornecendo uma visão realista das funcionalidades e interações necessárias para o gerenciamento de pacientes e suporte hospitalar. Cada tela foi projetada com atenção às necessidades específicas de médicos e enfermeiros, priorizando a eficiência no acesso e atualização de informações dos pacientes. Esse guia visual serve como uma referência crucial para validar o design e alinhar todos os componentes críticos do sistema com os objetivos funcionais e expectativas dos usuários antes da implementação final.
